## 0. 极简测试部署（推荐先做）

**目标**：尽量少步骤、**不必在服务器安装 Rust**，先把 API 跑起来对外测。

| 步骤 | 做什么 |
|------|--------|
| 1 | 一台 Linux VPS，安装 **Docker** + **Docker Compose V2** |
| 2 | 克隆仓库，`cd polymarket-account-analyzer` |
| 3 | 执行 **`docker compose up --build -d`**（使用默认 **`docker-compose.yml`**：Postgres + `analyzer` 容器） |
| 4 | 安全组 / 防火墙 **放行 TCP 3000** |
| 5 | 访问 **`http://<VPS公网IP>:3000/analyze/0x你的钱包`**；在服务器本机可 `curl http://127.0.0.1:3000/...` |

容器内已配置 **`DATABASE_URL`** 指向 compose 里的 `postgres`，一般**无需**再拷 `config.toml` 即可用缓存与 raw（与主 README「Docker Compose 部署」一致）。

**绑定域名 `api.forevex.trade`（可选）**

- 在 DNS 中为 **`forevex.trade`** 添加主机记录 **`api`** → **A** → VPS IPv4（有 IPv6 可加 **AAAA**）。
- 检查：`dig +short api.forevex.trade A`
- 仍走 **3000** 端口：`http://api.forevex.trade:3000/analyze/...`（记得放行 3000）。

**要上 HTTPS（仍很简单）**：在宿主机安装 **Caddy**，`Caddyfile`：

```caddyfile
api.forevex.trade {
    reverse_proxy 127.0.0.1:3000
}
```

`systemctl reload caddy` 后自动 **Let’s Encrypt**，对外：**`https://api.forevex.trade/analyze/{wallet}`**。浏览器跨域需配置 **`PAA_CORS_ORIGINS`** / `cors_allowed_origins`（见 **配置** 页与主 README）。

## 1. 进阶架构（Postgres 仅 Docker + 宿主机 Rust）

| 组件 | 运行方式 | 说明 |
|------|----------|------|
| **PostgreSQL** | **`docker-compose.postgres-only.yml`** | `5432` 只绑定 **127.0.0.1** |
| **Rust API** | 宿主机 `cargo build --release` | 监听 **127.0.0.1:3000** |
| **HTTPS** | **Caddy** / **Nginx** | 反代到 `127.0.0.1:3000`，域名 **`api.forevex.trade`** |

适用于：要在宿主机直接改代码/调试二进制、又不想在容器里跑 Rust。

## 2. 准备服务器（进阶）

- 已开放 **22**（SSH）、**80**、**443**（及测试阶段若直连 API 则需 **3000**）。
- 安装 **Docker** 与 **Docker Compose V2**。
- **进阶路径**另需 **Rust** 工具链（或上传已编译的 Linux 二进制）。

## 3. DNS：`api.forevex.trade`

| 类型 | 名称（常见面板） | 值 |
|------|------------------|-----|
| **A** | `api`（相对 `forevex.trade`） | VPS 公网 IPv4 |
| **AAAA**（可选） | `api` | VPS IPv6 |

```bash

```

## 4. 仅启动 PostgreSQL（Docker）

```bash
cd polymarket-account-analyzer
docker compose -f docker-compose.postgres-only.yml up -d
```

**生产请修改** compose 内 `POSTGRES_PASSWORD`，并设置：

```bash
export DATABASE_URL="postgres://postgres:你的强口令@127.0.0.1:5432/polymarket_analyzer"
```

---

## 5. 构建并运行 Rust API（宿主机）

```bash
cd polymarket-account-analyzer
cargo build --release
export DATABASE_URL="postgres://postgres:你的强口令@127.0.0.1:5432/polymarket_analyzer"
export RUST_LOG=info
./target/release/polymarket-account-analyzer serve --bind 127.0.0.1:3000
```

验证：`curl -sS "http://127.0.0.1:3000/analyze/0x你的钱包地址" | head -c 200`

### 跨平台编译（在 Mac 上构建 Linux 二进制）

```bash
rustup target add x86_64-unknown-linux-gnu
cargo build --release --target x86_64-unknown-linux-gnu
```

产物：`target/x86_64-unknown-linux-gnu/release/polymarket-account-analyzer`。上传到服务器后需 **glibc 兼容**；失败则在服务器本地 `cargo build`。

## 6. systemd 常驻（示例）

创建 `/etc/systemd/system/polymarket-analyzer.service`：

```ini
[Unit]
Description=Polymarket Account Analyzer API
After=docker.service network-online.target
Wants=network-online.target

[Service]
Type=simple
User=YOUR_LINUX_USER
WorkingDirectory=/opt/forevex/polymarket-account-analyzer
Environment=DATABASE_URL=postgres://postgres:YOUR_DB_PASSWORD@127.0.0.1:5432/polymarket_analyzer
Environment=RUST_LOG=info
ExecStart=/opt/forevex/polymarket-account-analyzer/polymarket-account-analyzer serve --bind 127.0.0.1:3000
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now polymarket-analyzer
```

勿将含密码的 unit 提交到 Git。

## 9. 安全清单

- 修改默认数据库口令；**极简 Docker** 方案下 Postgres **未**映射到宿主机 5432（与默认 `docker-compose.yml` 一致），但仍要改镜像内默认密码再上生产。
- **Postgres-only** 方案：仅 **`127.0.0.1:5432`**，勿 `0.0.0.0:5432` 对公网。
- 进阶反代：API 监听 **127.0.0.1**，由 Caddy/Nginx 对外 TLS。
- **极简测试**若用 `http://IP:3000`，评估是否仅内网/短期开放 3000。

## 10. 方式对比

| 方式 | Compose 文件 | 适用 |
|------|----------------|------|
| **一键 Postgres + API（极简）** | **`docker-compose.yml`** | 最快联调、测试 |
| Postgres only + 宿主机 Rust | **`docker-compose.postgres-only.yml`** | 本机调试 Rust、自定义 systemd |

更多 `PAA_*` 与 TOML 见 **配置** 页面与仓库 **README**。
